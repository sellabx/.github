# 安全策略

sellabx 构建用于业务关键场景的可扩展工具与基础设施。
我们严肃对待安全问题报告，并感谢负责任披露。

## 支持版本

我们仅为仍在积极维护的代码提供安全更新。

| 版本 | 是否支持 |
| --- | --- |
| main | 是 |
| 旧版本或不再维护的分支 | 否 |

## 漏洞报告方式

请勿在公开 issue 中报告安全漏洞。

请通过以下私密渠道之一提交：

- 邮箱：security@sellabx.com
- GitHub Security Advisory：使用本仓库的私有漏洞报告功能

请尽可能提供完整信息：

- 受影响组件及版本/提交
- 漏洞类型与影响
- 复现步骤或 PoC
- 可能的利用条件
- 建议修复方案（如有）

## 披露流程

1. 我们将在 72 小时内确认收悉。
2. 我们进行分级与严重性评估。
3. 我们协调修复并完成验证。
4. 我们在适当时发布修复并同步披露说明。

## 范围说明

在范围内：

- 代码执行类漏洞
- 认证/授权绕过
- 权限提升
- 敏感数据泄露
- 具有实际影响的依赖漏洞
- 具有安全影响的基础设施配置错误

不在范围内：

- 无实际影响、不可利用的理论问题
- 未证明风险的低价值最佳实践建议
- 不受支持版本中的漏洞

## 安全港原则

我们支持善意的安全研究，并在以下前提下不追究法律责任：

- 避免侵犯隐私与影响服务可用性
- 仅在你拥有的账号或被授权环境中测试
- 及时报告并给予我们合理修复时间

## 安全更新发布

安全修复将通过常规发布渠道交付，并在发布说明或安全公告中记录。
