# Support

sellabx is a tech lab building scalable tools and infrastructure for business-critical systems.

## 1. Support Channels

- GitHub Discussions: general usage questions, architecture discussions, and best practices
- GitHub Issues: bugs, feature requests, and reproducible technical problems
- Email: support@sellabx.com for account or organization-level support requests
- Security reports: security@sellabx.com (do not post security issues publicly)

## 2. Response Time Expectations

- Bug reports: 2-3 business days
- Feature requests: 3-5 business days
- Questions in Discussions: 2-4 business days
- Security reports: acknowledgement within 72 hours

Response times are targets, not guarantees, and may vary with severity and complexity.

## 3. Issue Prioritization

We prioritize requests based on impact and urgency:

- Highest priority: production-impacting bugs, security risks, and reliability regressions
- Medium priority: functional defects with workarounds and high-value improvements
- Lower priority: non-blocking enhancements, documentation gaps, and exploratory requests

Issue type guidance:

- Bug: incorrect or broken behavior
- Feature: new capability or meaningful enhancement
- Question: clarification on usage, setup, or expected behavior

## 4. Unsupported Scenarios

- Requests without reproducible details (steps, environment, or logs)
- Support for unmaintained forks or heavily modified private deployments
- Requests to troubleshoot unrelated third-party systems outside project scope
- Security disclosures posted publicly instead of private reporting channels

## Before You Open a Request

- Search existing issues and discussions first.
- Include environment details (OS, runtime, version/commit SHA).
- Share logs, error messages, and clear reproduction steps when possible.
