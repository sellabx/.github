# 支持说明

sellabx 是一个面向业务关键系统构建可扩展工具与基础设施的技术实验室。

## 1. 支持渠道

- GitHub Discussions：一般使用问题、架构讨论与最佳实践
- GitHub Issues：缺陷、功能请求与可复现技术问题
- 邮箱：support@sellabx.com（账号或组织级支持请求）
- 安全问题：security@sellabx.com（请勿公开披露）

## 2. 响应时效预期

- 缺陷报告：2-3 个工作日
- 功能请求：3-5 个工作日
- Discussions 提问：2-4 个工作日
- 安全报告：72 小时内确认收悉

以上为目标响应时效，不构成严格 SLA，实际时间会随问题严重度与复杂度变化。

## 3. 问题优先级

我们基于影响范围与紧急程度进行优先级排序：

- 最高优先级：生产影响缺陷、安全风险、稳定性回归
- 中优先级：存在替代方案但影响明显的功能问题
- 低优先级：非阻塞增强、文档缺口、探索性请求

问题类型指引：

- Bug：行为错误或功能损坏
- Feature：新增能力或有价值的功能增强
- Question：关于使用、配置或预期行为的咨询

## 4. 不支持场景

- 缺少可复现信息（步骤、环境、日志）的请求
- 对未维护分支、长期分叉或重度私有改造版本的排障支持
- 对项目范围外第三方系统的故障排查请求
- 以公开 issue 方式提交安全漏洞

## 提交前建议

- 先搜索现有 issue 与 discussions。
- 提供环境信息（OS、运行时、版本或提交 SHA）。
- 尽可能提供日志、报错信息与明确复现步骤。
