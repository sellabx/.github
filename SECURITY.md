# Security Policy

sellabx builds scalable tools and infrastructure used in business-critical environments.
We take security reports seriously and appreciate responsible disclosure.

## Supported Versions

Security updates are provided for actively maintained code.

| Version | Supported |
| --- | --- |
| main | Yes |
| Legacy or unmaintained branches | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use one of the following private channels:

- Email: security@sellabx.com
- GitHub Security Advisory: use private vulnerability reporting for this repository

Include as much detail as possible:

- Affected component and version/commit
- Vulnerability type and impact
- Reproduction steps or proof of concept
- Potential exploit conditions
- Suggested remediation (if available)

## Disclosure Process

1. We acknowledge receipt within 72 hours.
2. We triage and assess severity.
3. We coordinate remediation and validation.
4. We release a fix and publish disclosure notes when appropriate.

## Scope

In scope:

- Code execution vulnerabilities
- Authentication/authorization bypass
- Privilege escalation
- Sensitive data exposure
- Dependency vulnerabilities with practical impact
- Infrastructure misconfigurations with security impact

Out of scope:

- Non-exploitable theoretical issues without impact
- Low-value best-practice suggestions without a demonstrated risk
- Vulnerabilities in unsupported versions

## Safe Harbor

We support good-faith security research and will not pursue legal action for:

- Avoiding privacy violations and service disruption
- Testing only against your own accounts or permitted environments
- Promptly reporting findings and giving us reasonable time to remediate

## Security Updates

Security fixes will be delivered through normal release channels and documented in release notes or advisories.
